# Project Structure

## Root Configuration

- `vite.config.ts` - Vite configuration with plugins
- `tsconfig.json` - TypeScript compiler options with path aliases
- `eslint.config.js` - ESLint flat config
- `.prettierrc.json` - Prettier formatting rules
- `tailwind.config.ts` - Tailwind CSS configuration
- `docker-compose.yml` / `Dockerfile` - Container deployment setup

## Source Directory (`src/`)

```
src/
├── assets/          # Static assets (images, SVGs)
├── components/      # Reusable React components
│   ├── ui/         # shadcn/ui components
│   └── index.ts    # Component exports
├── constants/       # Application constants
├── data/           # Static data and mock data
├── helpers/        # Helper functions
├── hooks/          # Custom React hooks
├── pages/          # Route components (file-based routing)
├── store/          # State management
├── types/          # TypeScript type definitions
├── utils/          # Utility functions
│   └── cn.ts      # Tailwind class merging utility
├── index.css       # Global styles and Tailwind directives
└── main.tsx        # Application entry point
```

## Import Conventions

- **Path Alias**: Use `@/` for imports from `src/` directory
  - Example: `import { Button } from '@/components/ui/button'`
- **Auto-imports**: React hooks (useState, useEffect, etc.) and React Router hooks are auto-imported
- **SVG Components**: Import SVGs with `?react` query
  - Example: `import Logo from '@/assets/logo.svg?react'`

## Routing

- File-based routing powered by `vite-plugin-pages`
- Route files located in `src/pages/`
- Each `.tsx` file in `pages/` becomes a route automatically
- Special files:
  - `index.tsx` - Root route (`/`)
  - `NotFound.tsx` - 404 page
  - `RootErrorBoundary.tsx` - Error boundary

## Component Organization

- UI components from shadcn/ui go in `src/components/ui/`
- Custom components go in `src/components/`
- Export components through `index.ts` for cleaner imports
- Each component folder should have its own index file when applicable

## Configuration Files

- `configs/fonts.config.ts` - Google Fonts configuration
- `auto-imports.d.ts` - Auto-generated type definitions for auto-imports (do not edit manually)
- `components.json` - shadcn/ui configuration

## Ignored Files

- `auto-imports.d.ts` - Generated by unplugin-auto-import
- `node_modules/`, `dist/`, `.output/` - Build artifacts
- `.nitro/` - Nitro cache
